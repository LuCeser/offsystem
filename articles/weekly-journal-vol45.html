博客系统新范式

这周，我在使用 Claude Code 写作时，突然意识到：
博客系统的逻辑，正在被彻底改写。

以往我们写博客，要先选择框架（Hexo、Hugo、Astro），再挑主题、配置参数、编译部署。那是一种"人去适应系统"的方式——我们学习它的语法、遵守它的约定，只为让文字有一个体面的出口。
但这一次，我只输入一句话："我想要一个像《纽约时报》那样的现代、简洁网站。" Claude Code 便自动生成了页面结构、排版风格与样式代码。我不再需要下载主题、不再关心模板细节，只需继续写作。

这一刻，我感受到了一种"表达范式"的跃迁：
AI 不再是辅助工具，而是生成媒介。系统退到幕后，表达成为前台。
写博客从"配置工程"回到了"思维呈现"。
技术的隐退，反而让思想的形状变得更加清晰。

我开始思考：未来的博客，可能不再是一个"系统"，而是一种"接口"。
作者只需描述想要的表达形态，AI 即时完成排版与发布。页面不再是手工搭建的结果，而是语义生成的产物。
这意味着：技术门槛正在隐形消失，而表达的门槛正在重新建立。

或许博客的未来，不在系统本身，而在思想被呈现的速度。

本周阅读 | 《杨家将传说到底有几分真》（《文明之旅》）

本周听到一期印象深刻的《文明之旅》，讲的是"杨家将传说"的成形过程。
在历史中，杨家将的真实事迹寥寥，但后世文学、戏曲、影视不断"再创作"，形成了一个更具象征意义的"定本"。
罗振宇提到"口述"与"定本"的区别让我印象深刻。
"口述"意味着故事在流动中被不断解释与吸收，而"定本"则意味着权威的确立、叙事的冻结。

这让我想到孔子"述而不作"的哲学。孔子选择"不作"，并非出于谦虚，而是一种教育上的自觉。
他讲"因材施教"，强调知识与人的匹配性。如果他将思想"作"成体系，就等于为后人划定了唯一的路径；而"述而不作"让思想保持开放，使后来的弟子能够在不同语境中重新生成意义。

一旦《论语》被固定为"孔子之书"，教育就被封装进文本之中。千年之后的教学只能从书中去找孔子的答案，而非创造新的问法。这正是"定本"的代价：知识得到了保存，却失去了再生的可能。

联想到《智人之上》提到：
"文字与档案系统改变了文明的走向。"
当知识被固定在文字中，它获得了稳定性，却也失去了再解释的自由。文明的演化，其实是在"记录"与"再创作"之间寻找平衡。

本周阅读 | 生成式模型的 AI 味

这周还读到一篇文章 Why do AI models use so many em-dashes?，
探讨为什么生成式 AI 的文本中充满了破折号。

我在与 ChatGPT 对话中也深有体会：它的标题常常是"主题：说明"的结构，那种四平八稳，看似有道理但没有任何棱角的语言风格，正是一种显著的"AI味"。
模型的解释是：

生成式模型以英文语料为主，生成中文时延续英文句法节奏，保留破折号与解释句结构。

在英文写作中，破折号（—）常用于插入语或强调，是自然的"呼吸点"；而中文的破折号更少见、更具情绪色彩。当模型将英文语法节奏映射到中文，它创造出一种"中性流畅"的节奏感——听起来顺滑，却削弱了语言的张力。

本质上，"AI味"是一种去语境化的均值语言。
它来自模型在概率空间中对语料的平均化处理：生成最安全、最可能、也最无锋芒的表达。人类的语言依赖语境、意图与偏好，而模型的语言依赖分布、相似与概率。AI 写出的"句子"流畅，却往往没有"气息"。

就像我这篇文字一样，如果没有我去调整，也会是一篇毫无风格的"AI生成"博客。随着使用AI的人越来越多，估计会看到越来越多"AI味"的语言。

这就是本周的分享，希望对你有所启发。如果你有任何想法或建议，欢迎与我交流。
